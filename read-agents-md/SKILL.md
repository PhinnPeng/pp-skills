# Read AGENTS.md

## 目标

读取当前上下文适用的 **AGENTS.md**，将其作为后续行为的约束基线；对照近期对话（最多 3 轮）检查是否偏离，并修正不符合项。

## 何时使用

- 用户 @ 本技能，或提到 AGENTS.md / 项目代理指令
- 进入新项目或切换工作区，且存在 AGENTS.md
- 发现回复或操作可能与项目约定冲突，需要重新对齐

**不适用**：项目无 AGENTS.md，且用户未提供等效指令文件。

## 工作流

```
Task Progress:
- [ ] 1. 定位适用的 AGENTS.md
- [ ] 2. 读取并提炼约束
- [ ] 3. 审查近期对话（≤3 轮）
- [ ] 4. 修正偏差并确认后续遵循
```

### 1. 定位 AGENTS.md

按优先级查找（命中即读，可叠加多层）：

| 优先级 | 路径 | 说明 |
|--------|------|------|
| 1 | 当前工作文件所在目录及祖先目录 | 子目录 AGENTS.md 覆盖父级 |
| 2 | 工作区根目录 `AGENTS.md` | 项目全局指令 |
| 3 | `~/.codex/AGENTS.md` | 用户级全局（Codex/Cursor 等） |
| 4 | 用户明确指定的路径 | 以用户指示为准 |

使用 Glob 或 Read 工具实际读取，**不要凭记忆假设内容**。

**未找到时**：告知用户缺失位置，询问是否有 `.cursor/rules/`、`CLAUDE.md` 等等效文件；无则跳过审查，不虚构规则。

### 2. 读取并提炼

读取全文后，在内心（或简短对内摘要）整理为可执行约束：

- **命令**：构建、测试、格式化、部署的固定写法
- **环境**：版本、包管理器、URL、密钥占位符
- **风格**：命名、目录、禁止模式
- **流程**：PR 目标分支、提交规范、验证步骤
- **边界**：禁止操作、必须询问的事项

多层 AGENTS.md 冲突时：**更具体（更深目录）优先**；同级冲突则标注并询问用户。

### 3. 审查近期对话

回溯**最多 3 轮**用户—助手往来，逐项对照 AGENTS.md：

| 维度 | 检查点 |
|------|--------|
| 命令 | 是否用了文件禁止或遗漏的包装命令（如 `uv run`） |
| 范围 | 改动是否超出允许范围或未遵循小步提交 |
| 风格 | 代码/文案是否符合项目约定 |
| 流程 | 是否跳过规定的验证、分支或审批步骤 |
| 语言 | 用户规则与 AGENTS.md 冲突时，**以更严格或用户显式要求为准** |

只审查可核实的行为与输出；不对用户未发生的操作做臆测。

### 4. 修正与确认

对每项偏差：

1. **说明**：哪条 AGENTS.md 规则被违反
2. **修正**：重做错误步骤、补全遗漏验证、调整后续方案
3. **承诺**：后续在本会话中严格遵循已读取的 AGENTS.md

无需修正时，用 1–2 句确认已对齐即可。

## 输出格式

向用户汇报时使用中文（命令原始输出可保留英文）：

```markdown
## AGENTS.md 对齐

**来源**：`path/to/AGENTS.md`（及嵌套文件，如有）

**关键约束**（3–7 条要点）：
- ...

**近期对话审查**（≤3 轮）：
- ✅ 符合 / ⚠️ 偏差：… → 已修正：…

**后续遵循**：已内化上述约束，后续操作将据此执行。
```

## 原则

- **先读后做**：涉及项目命令、结构、流程时，先完成读取再行动
- **证据优先**：引用 AGENTS.md 原文或章节标题，避免泛泛而谈
- **最小修正**：只改与 AGENTS.md 冲突的部分，不借机扩大重构范围
- **全程中文**：与用户交互使用简体中文；文件路径、命令、标识符保持原文
